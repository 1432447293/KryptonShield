# 氪盾 · KryptonShield

> 面向超算时代的个人数据要塞 —— 基于 Argon2 + AES-256-GCM 的抗暴力破解加密工具

[![Python](https://img.shields.io/badge/Python-3.7+-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## 📖 简介

**氪盾（KryptonShield）** 是一款开源的离线加密工具，旨在保护个人文件与文件夹免受普通超级计算机及 GPU 集群的暴力破解攻击。它基于公开密码学标准构建，核心目标是为敏感的个人信息提供可靠的数据保护。

与普通 ZIP 加密或简易 AES 实现不同，氪盾采用了 **内存硬密钥派生函数 Argon2id**。通过在每次解密尝试时强制消耗大量内存与计算资源，使攻击者无法利用低成本硬件进行大规模并行密码猜测，从而在经济和技术层面大幅提升暴力破解的难度。

### ✨ 核心特性

- 🔐 **军用级加密标准**：采用 AES-256-GCM 认证加密模式，同时保证数据的机密性、完整性与防篡改性。
- 🧠 **内存硬抗破解**：密钥派生使用 Argon2id 算法，默认内存消耗为 512 MiB，能够有效压制 GPU/ASIC/FPGA 破解集群的攻击。
- 📁 **文件夹一键打包加密**：自动将整个目录压缩为 ZIP 格式再进行加密，并在解密时智能还原文件夹结构。
- 🌐 **完全离线运行**：不依赖任何网络连接，不收集任何数据，不留任何后门。密码是唯一凭证。
- ⚙️ **可调安全参数**：高级用户可根据自身需求调整 Argon2 的时间成本与内存成本，以平衡安全性与运行性能。

## 🔧 技术原理

### 1. 密钥派生 —— Argon2id

Argon2 是 2015 年密码哈希竞赛（PHC）的冠军算法，被公认为当前最先进的密码哈希/密钥派生函数。本工具选用其 **Argon2id** 变体，兼具抗侧信道攻击与抗时间-内存权衡（TMTO）攻击的能力。

**默认参数：**

| 参数 | 值 | 描述 |
|------|-----|------|
| 时间成本 (Time Cost) | 4 | 迭代次数，控制计算负载 |
| 内存成本 (Memory Cost) | 512 MiB | **关键参数**，强制占用高内存 |
| 并行度 (Parallelism) | 2 | 并行线程数量 |
| 盐值长度 (Salt Length) | 16 字节 | 随机生成，防止彩虹表攻击 |
| 输出长度 (Hash Length) | 32 字节 | 经 SHA256 二次哈希后作为 AES 密钥 |

### 2. 对称加密 —— AES-256-GCM

- **AES-256**：256 位密钥长度，符合美国国家安全局（NSA）最高机密级加密标准。
- **GCM 模式**：Galois/Counter 模式，属于 **带关联数据的认证加密（AEAD）** 方案。加密过程中会生成一个认证标签（Tag），任何对密文的修改都会导致解密验证失败并抛出异常。

### 3. 加密数据结构

每个加密文件（`.enc` / `.zip.enc`）的内部二进制布局如下：

+----------------+----------------+----------------+----------------------+
|      Salt      |     Nonce      |      Tag       |      Ciphertext      |
|   (16 字节)    |   (12 字节)    |   (16 字节)    |       (变长)         |
+----------------+----------------+----------------+----------------------+

- **Salt**：Argon2 密钥派生所用的随机盐值。
- **Nonce**：AES-GCM 使用的初始化向量，每次加密随机生成。
- **Tag**：GCM 模式生成的认证标签，用于完整性验证。
- **Ciphertext**：原始明文数据加密后的密文。

## 🚀 快速开始

### 环境要求

- Python 3.7 或更高版本
- pip 包管理器

### 安装依赖

```bash
pip install argon2-cffi pycryptodome
```

### 下载源码

将 `kryptonshield.py` 下载到本地，或直接复制代码保存。

### 运行程序

```bash
python kryptonshield.py
```

运行后将显示交互式菜单：

```
======================================================================
          高强度文件/文件夹加密工具 (Argon2 + AES-256-GCM)
======================================================================
请选择操作：
  1. 加密单个文件
  2. 解密文件（自动识别并解压文件夹）
  3. 加密整个文件夹（自动打包为 ZIP）
======================================================================
请输入选项 (1/2/3)：
```

## ⚙️ 高级配置

有特定安全需求的用户可以直接修改 `kryptonshield.py` 开头的参数：

```python
# --- 配置参数（可根据安全需求调整） ---
KEY_LENGTH = 32
ARGON2_TIME_COST = 4          # 迭代次数；增加可提升抗计算攻击能力
ARGON2_MEMORY_COST = 512 * 1024 # 内存成本（512 MB）；增加可大幅提升抗 GPU/ASIC 攻击能力
ARGON2_PARALLELISM = 2        # 并行度
ARGON2_HASH_LEN = 32
ARGON2_SALT_LEN = 16
```

| 参数调整 | 对合法用户的影响 | 对攻击者的影响 |
|----------|------------------|----------------|
| 增大 `MEMORY_COST` | 加解密时占用更多内存，速度变慢 | 破解设备需配备更大显存/内存，成本指数级上升 |
| 增大 `TIME_COST` | 加解密耗时线性增加 | 单次密码猜测耗时线性增加 |
| 减小参数 | 提升在低配设备上的运行流畅度 | 削弱抗破解能力，请谨慎操作 |

> ⚠️ **注意**：修改参数后，加密的文件只能由使用 **相同参数** 的程序解密。建议确定一套固定参数并长期使用。

## 🛡️ 安全说明与免责声明

### 密码安全
- **密码是唯一的凭证**。本工具 **不会存储任何密码或恢复密钥**。一旦密码遗忘，加密数据将 **永久无法恢复**。
- 强烈建议使用 **高熵密码**（例如随机生成的 12 位以上混合字符），并将其存储在密码管理器中。

### 使用限制
- 本工具仅限用于合法的个人数据保护。严禁用于任何非法活动。
- 请妥善保管加密后的文件；文件损坏（如传输错误、磁盘坏道）可能导致数据永久丢失。

### 威胁模型
- **可有效防御**：由普通超级计算集群、GPU 破解农场及专用 ASIC 硬件发起的离线暴力破解攻击。
- **无法防御**：键盘记录器、内存抓取、冷启动攻击、社会工程学、法律强制要求等非密码学攻击手段。

使用者须遵守所在国家/地区的法律法规。开发者对任何滥用本工具的行为不承担任何责任。
